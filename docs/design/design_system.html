Folder structure

— 04.02 · astro + react split
src/ ├── pages/ // Astro routes — SSG by default │ ├── index.astro │ ├── about.astro │ ├── projects.astro │ ├── contact.astro │ └── blog/ │ ├── index.astro // list │ └── [slug].astro // post from content collection ├── layouts/ │ ├── BaseLayout.astro // <head>, fonts, theme │ └── PostLayout.astro ├── components/ │ ├── static/ // .astro — no JS shipped │ │ ├── Header.astro │ │ ├── Footer.astro │ │ ├── ArticleCard.astro │ │ ├── ProjectCard.astro │ │ └── Button.astro │ └── islands/ // .tsx — hydrated only where used │ ├── SearchBar.tsx │ ├── ContactForm.tsx │ ├── SubscribeModal.tsx │ ├── ProjectFilter.tsx │ └── Carousel.tsx ├── content/ │ ├── blog/ // MDX posts │ └── projects/ // .json manifests ├── styles/ │ ├── tokens.css // :root vars, single source of truth │ └── global.css └── assets/ ├── icons/ // individual .svg, inlined at build ├── fonts/ // self-hosted woff2 └── img/ // .webp / .avif, optimized
Rule of thumb. Ship in .astro unless you need: local state, focus management, keyboard choreography, or runtime data. Islands hydrate with client:visible by default, client:idle for non-critical.

Coding standard

— 04.03 · tailwind + bem escape
Utility-first by default
Tailwind drives layout, spacing, and simple color/state. Config extends directly from tokens.css via theme.extend.colors — no duplicated values. Prefer bg-bg-alt border border-border rounded-md over bespoke classes.
BEM escape hatch
When a component has >4 states or coordinated children, drop to BEM in a scoped <style> block: .chatbar, .chatbar__input, .chatbar--compact. Never mix utility + BEM on the same element.
Naming conventions
Files: PascalCase.astro / .tsx. CSS vars: --kebab-case. Data attrs: data-scope-thing. Event handlers: onThing, never handleThing inside JSX.
No magic values anywhere
Every color, radius, shadow, spacing value must resolve to a token. If you're typing #, px outside of the scale, or rgba(, you're adding debt — open an issue to add a token first.
Dark mode is a contract
Every token must have a [data-theme="dark"] counterpart. Never hardcode #fff / #000; use --surface and --ink. Toggle is attribute-based so Astro can server-render the correct theme from a cookie.
